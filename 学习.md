# 小米 UniApp 项目学习文档

## 一、项目概述

本项目是一个**小米风格的企业级后台管理系统**，采用前后端分离架构，前端基于 uni-app + Vue 3 + TypeScript 实现跨平台（H5 / 微信小程序 / App）部署，后端基于 FastAPI + SQLAlchemy + MySQL 提供 RESTful API 服务。

| 维度 | 技术选型 |
|------|---------|
| 前端框架 | uni-app + Vue 3 (Composition API) + TypeScript |
| 构建工具 | Vite 5 |
| 后端框架 | FastAPI 0.104 |
| ORM | SQLAlchemy 2.0 |
| 数据库 | MySQL (PyMySQL 驱动) |
| 认证方案 | JWT (python-jose + passlib/bcrypt) |
| 部署方式 | Docker + docker-compose / Uvicorn |

### 3.2 认证体系

**services/auth.py** — JWT 认证服务

```
登录流程：
1. 用户提交 username + password
2. 后端用 bcrypt 验证密码
3. 生成 JWT Token（payload 含 sub=用户ID, exp=过期时间）
4. 返回 access_token 给前端

请求鉴权流程：
1. 前端在请求头携带 Authorization: Bearer <token>
2. get_current_user() 依赖注入解析 Token
3. 提取 user_id 查询数据库
4. 检查用户状态（是否禁用）
5. 验证失败返回 401/403
```

## 四、前端架构详解

### 4.1 请求封装

**api/request.ts** — 统一 HTTP 请求层

```
核心机制：
1. BASE_URL = '/uniapp-api'，所有请求自动拼接
2. 自动从 uni.getStorageSync('token') 读取 Token
3. 自动添加 Authorization: Bearer <token> 请求头
4. 统一响应处理：
   - HTTP 200 + code 200 → 成功
   - code 401 / HTTP 401 → 清除 Token，跳转登录页
   - 其他错误 → Toast 提示错误信息

便捷方法：
  get<T>(url, data)   → GET 请求
  post<T>(url, data)  → POST 请求
  put<T>(url, data)   → PUT 请求
  del<T>(url, data)   → DELETE 请求
```

**请求链路：**
```
前端 api/*.ts 调用 get/post/put/del
  → request.ts 拼接 /uniapp-api 前缀 + 注入 Bearer Token
    → Vite 开发代理: /uniapp-api → http://localhost:8022/api
      → Nginx 生产代理: /uniapp-api → http://后端IP:8022/api
        → FastAPI 路由处理
```

### 4.5 页面功能详解

## 七、关键设计模式

### 7.1 统一响应格式

所有后端接口返回统一格式：
```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 7.2 认证中间件

- 使用 FastAPI 的 `Depends(get_current_user)` 依赖注入
- 从 `Authorization: Bearer <token>` 提取 JWT
- Token 中 `sub` 字段存储用户 ID
- 前端 `request.ts` 自动处理 401 跳转登录

### 7.3 跨内容类型统一设计

娱乐中心的收藏、历史、分类等表使用 `content_type` 字段区分内容类型（game/music/video/novel），实现一套代码服务四种内容。

### 7.4 前端多端适配

相机、分享等功能根据平台差异使用条件编译：
```typescript
// #ifdef H5
navigator.mediaDevices.getUserMedia()
// #endif
// #ifdef APP-PLUS
plus.camera.getCamera()
// #endif
// #ifdef MP-WEIXIN
uni.createCameraContext()
// #endif
```

### 7.5 数据缓存策略

- 天气数据：后端内存缓存 30 分钟
- 用户 Token：前端 uni.setStorageSync 持久化
- 推荐结果：前端 Storage 临时传递到结果页
